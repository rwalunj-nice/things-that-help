## 🚀 Getting Started

1. **Review the Quick Benchmark** — Start with [`github-quick-benchmark.md`](github-quick-benchmark.md) to understand which metrics matter most when evaluating GitHub Copilot.
2. **Pick your tier** — Focus on **Tier 1 (Core Signals)** first before diving into ROI or context metrics.
3. **Set baselines** — Use the "Good Looks Like" column to establish targets, then track trends over time within your team.

---

## 📊 Metric Tiers at a Glance

The benchmark organizes Copilot metrics into three tiers:

- **🥇 Tier 1 — Core Signals**: % Code Acceptance, Code Acceptance Count, and Agent Adoption. These tell you whether Copilot is relevant, trusted, and evolving in usage.
- **🥈 Tier 2 — ROI Signals**: Request Efficiency and Premium Requests Count. These help justify spend and catch budget outliers.
- **🥉 Tier 3 — Context Signals**: Initiated Interactions, Suggestion Efficiency, Lines of Code metrics, and Code Generation Count. Use these to diagnose anomalies — never as standalone success indicators.

> **Tip:** Use the ⚡ Quick Decision Map in the benchmark file to jump straight to the metrics that answer your specific question.
